Quando usamos o KEY mapeado para o clock, o comportamento era como esperado para todos os modos do registrador de deslocamento universal. Entretanto, ao mapear o SW para o clock, o comportamento para os modos shift left e shift right estava incorreto, pois toda vez que mudávamos o switch de 0 para 1, ele copiava o bit da entrada serial para todos os bits da saída paralela, sobrescrevendo os dados anteriores.
Esse comportamento é devido ao ruído dos switches, que não trocam o sinal de 0 para 1 de forma discreta e apresentam várias oscilaçôes no processo. Cada oscilação do ruído é interpretada como um clock de subida e faz com que o dado da entrada serial seja copiado para todos os bits da saída paralela. Por outro lado, o key tem um circuito capaz de eliminar esse ruído tornando-o a escolha certa para mapear o clock de entrada.
